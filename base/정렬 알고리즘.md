# 정렬 알고리즘

# 대표적인 정렬 알고리즘 비교

## 1. 시간복잡도 기준 분류

### **O(n²) 알고리즘 - 단순 정렬**

**버블 정렬 (Bubble Sort)**

- **핵심원리**: 인접한 두 원소를 비교하며 큰 값을 뒤로 보내는 과정을 반복
- **시간복잡도**: 최선/평균/최악 모두 O(n²)
- **공간복잡도**: O(1) - 제자리 정렬
- **안정성**: 안정 정렬
- **특징**: 구현이 가장 단순하지만 실용성이 매우 낮음

**선택 정렬 (Selection Sort)**

- **핵심원리**: 매번 최솟값(또는 최댓값)을 찾아서 맨 앞으로 이동
- **시간복잡도**: 최선/평균/최악 모두 O(n²)
- **공간복잡도**: O(1) - 제자리 정렬
- **안정성**: 불안정 정렬
- **특징**: 교환 횟수가 적어 교환 비용이 큰 경우 유리할 수 있음

**삽입 정렬 (Insertion Sort)**

- **핵심원리**: 정렬된 부분에 새 원소를 적절한 위치에 삽입
- **시간복잡도**: 최선 O(n), 평균/최악 O(n²)
- **공간복잡도**: O(1) - 제자리 정렬
- **안정성**: 안정 정렬
- **특징**:
    - **거의 정렬된 데이터에서는 매우 효율적** (최선 O(n))
    - 작은 데이터셋에서 빠름
    - 온라인 알고리즘 (데이터가 실시간으로 들어와도 정렬 가능)

---

### **O(n log n) 알고리즘 - 고급 정렬**

**병합 정렬 (Merge Sort)**

- **핵심원리**: 분할 정복 - 반으로 나누고, 각각 정렬 후 병합
- **시간복잡도**: 최선/평균/최악 모두 O(n log n) - **안정적인 성능**
- **공간복잡도**: O(n) - 추가 배열 필요
- **안정성**: 안정 정렬
- **특징**:
    - 성능이 일정하여 **최악의 경우에도 보장된 성능**
    - 연결 리스트 정렬에 적합
    - **외부 정렬**(디스크 기반)에 사용됨

**퀵 정렬 (Quick Sort)**

- **핵심원리**: 피벗을 기준으로 작은 값과 큰 값을 분할하여 재귀적 정렬
- **시간복잡도**:
    - 최선/평균 O(n log n)
    - **최악 O(n²)** - 이미 정렬된 경우, 피벗 선택이 나쁜 경우
- **공간복잡도**: O(log n) - 재귀 스택
- **안정성**: 불안정 정렬
- **특징**:
    - **평균적으로 가장 빠른 정렬** (캐시 효율성 높음)
    - 제자리 정렬로 구현 가능
    - 피벗 선택 전략이 성능에 큰 영향 (랜덤, 중간값 등)

**힙 정렬 (Heap Sort)**

- **핵심원리**: 힙 자료구조를 이용해 최댓값/최솟값을 반복적으로 추출
- **시간복잡도**: 최선/평균/최악 모두 O(n log n)
- **공간복잡도**: O(1) - 제자리 정렬
- **안정성**: 불안정 정렬
- **특징**:
    - 추가 메모리 없이 O(n log n) 보장
    - **우선순위 큐 구현**의 기반

---

### **특수한 경우의 정렬**

**카운팅 정렬 (Counting Sort)**

- **핵심원리**: 각 값의 등장 횟수를 세어서 정렬
- **시간복잡도**: O(n + k) (k는 데이터 범위)
- **공간복잡도**: O(k)
- **안정성**: 안정 정렬
- **제약조건**: **정수이며 범위가 제한적**일 때만 사용 가능
- **특징**: 조건이 맞으면 선형 시간에 정렬 가능

## 2. 핵심 선택 기준

| 상황 | 추천 알고리즘 | 이유 |
| --- | --- | --- |
| **일반적인 경우** | 퀵 정렬 | 평균 성능이 가장 빠름 |
| **최악의 경우 보장 필요** | 병합 정렬, 힙 정렬 | 항상 O(n log n) |
| **메모리 제약** | 힙 정렬, 퀵 정렬 | 제자리 정렬 |
| **안정성 필요** | 병합 정렬, 삽입 정렬 | 같은 값의 순서 유지 |
| **거의 정렬된 데이터** | 삽입 정렬 | O(n)에 근접 |
| **작은 데이터셋** | 삽입 정렬 | 단순하고 빠름 |
| **정수, 제한된 범위** | 카운팅 정렬 | O(n) 가능 |

## 3. 다른 개념과의 연결

**데이터베이스 조인 알고리즘**

- **정렬 병합 조인**: 두 테이블을 정렬 후 병합하는 방식으로, 병합 정렬의 병합 과정과 유사
- 비등가 조인(<, >, <=, >=)에서 효율적
- 시간복잡도: O(M log M + N log N)

**힙 자료구조**

- 힙 정렬은 힙의 삽입/삭제 O(log n) 특성을 활용
- **우선순위 큐**에서 최댓값/최솟값을 반복 추출하는 것이 힙 정렬의 원리
- 관련 개념: 다익스트라 알고리즘, CPU 스케줄링의 우선순위 큐

**분할 정복 (Divide and Conquer)**

- 병합 정렬, 퀵 정렬이 대표적인 분할 정복 알고리즘
- 재귀 호출 시 **스택 메모리** 사용 → 공간복잡도 고려

**캐시 지역성 (Cache Locality)**

- 퀵 정렬이 병합 정렬보다 빠른 이유 중 하나
- 메모리 접근 패턴이 연속적이어서 캐시 히트율 높음

## 4. 면접 꼬리질문 대비 포인트

**"왜 퀵 정렬이 평균적으로 가장 빠른가?"**
→ 캐시 지역성이 좋고, 제자리 정렬로 메모리 오버헤드가 적으며, 실제 데이터에서 피벗 선택이 나쁠 확률이 낮음

**"퀵 정렬의 최악의 경우를 어떻게 방지하나?"**
→ 랜덤 피벗 선택, 중간값(median-of-three) 사용, 인트로소트(Introsort)처럼 재귀 깊이 제한 후 힙 정렬로 전환

**"안정 정렬이 필요한 경우는?"**
→ 여러 기준으로 정렬할 때 (예: 먼저 점수로 정렬, 같은 점수는 이름 순 유지)

**"왜 Java/Python의 기본 정렬은 Timsort를 사용하나?"**
→ 삽입 정렬 + 병합 정렬의 하이브리드로, 실제 데이터(부분적으로 정렬된 경우 많음)에서 최적화됨

**"정렬 알고리즘을 직접 구현해야 하는 경우는?"**
→ 거의 없음. 표준 라이브러리 사용. 다만 **커스텀 비교 함수**를 작성하는 능력은 필수

---

## 5. 실무 관점

- **실무에서는 표준 라이브러리 사용** (Java: Arrays.sort, Collections.sort / Python: sorted, sort)
- 중요한 것은 **언제 정렬이 필요한지**, **정렬의 비용**을 인지하는 것
- 데이터가 이미 정렬되어 있다면 **이진 탐색** O(log n) 가능 → 자료구조 선택에 영향
